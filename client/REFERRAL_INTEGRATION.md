# Referral System Integration - Client Side

## Overview

The referral system is now integrated into the registration flow. Users can optionally enter a referral code during signup to receive rewards.

## How It Works

### 1. **Registration with Referral Code**

When a user registers, they can optionally enter a referral code in the format `XXXX-XXXX` (e.g., `ABCD-1234`).

**Features:**
- Optional field - users can skip it
- Auto-formatting: automatically adds hyphen after 4 characters
- Auto-uppercase: converts input to uppercase
- Validation: ensures correct format (4 chars, hyphen, 4 chars)
- Real-time feedback: shows reward message when valid format is entered

### 2. **Where Referral Codes Come From**

Each user automatically gets a unique referral code when they register. This code is:
- Generated by the backend
- Unique to each user
- In the format `XXXX-XXXX`
- Available in the user profile (`user.referral_code`)

### 3. **Reward Structure**

According to the backend documentation:

| Event | Referrer Reward | Referred User Reward |
|-------|----------------|---------------------|
| First order > ₹500 | ₹20 | ₹5 |

**Rules:**
- Rewards are granted when the referred user completes their **first order**
- Order value must be **greater than ₹500**
- Rewards are processed automatically by the backend
- Users can redeem their balance (minimum ₹120) on future orders

### 4. **User Flow**

```
1. New User Registration
   ↓
2. (Optional) Enter Referral Code from Friend
   ↓
3. Complete Registration & OTP Verification
   ↓
4. Place First Order (> ₹500)
   ↓
5. Order Completed
   ↓
6. Rewards Automatically Granted:
   - Referrer gets ₹20
   - New user gets ₹5
```

```typescript

```

### API Integration

```typescript
// Registration with referral code
const registerData = {
  email: formData.email,
  name: formData.fullName,
  password: formData.password,
  confirm_password: formData.confirmPassword,
  promo_code: formData.referralCode.trim().toUpperCase(), // Optional
};

await AuthService.register(registerData);
```

## Backend Integration

The backend handles:
- Validating referral codes
- Linking users to referrers
- Generating unique codes for new users
- Processing rewards automatically via Django signals
- Managing referral balances
- Handling redemptions

**Backend Endpoints:**
- `POST /api/authentication/register` - Register with optional `promo_code`
- `GET /api/authentication/user` - Get user profile with referral data
- `POST /api/inventory/create-order/` - Create order with optional redemption

## Notes

- Referral codes are **case-insensitive** (automatically converted to uppercase)
- Invalid codes are **rejected** during registration (user must fix or remove)
- Backend validates codes against the database
- Rewards are processed **automatically** when orders are completed
- Users can view their referral code and balance in their profile
